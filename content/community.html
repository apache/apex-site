Community

Mailing Lists

The following mailing lists are available for Apex.

  • users@apex.incubator.apache.org is a discussion forum for people who want to build applications and/or solutions using Apex. subscribe, unsubscribe, ponymail, nabble, archives
  • dev@apex.incubator.apache.org is for people who want to contribute code to Apex. Traffic on the list is generally high and includes JIRA notifications and code review comment. subscribe, unsubscribe, ponymail, nabble, archives
  • commits@apex.incubator.apache.org is for commit messages and patches to Apex. subscribe, unsubscribe, ponymail, nabble, archives
  • security@apex.incubator.apache.org is for reporting security related issues with Apex. It is a private list to discuss security related issues. Please post potential security vulnerabilities to this list so that they can be investigated and fixed before the vulnerabilities are published to the public. You can post to the list without subscribing to it.

Contributing

To learn more about how we use Github to accept contributions to the project, check out the contributing guidelines.

The Apex Project is made up of two repositories:

  • Apex Core - The core of the Apex platform.
  • Apex Malhar - Community-driven set of open-source "operators" and utilities for use in your Apex applications.
